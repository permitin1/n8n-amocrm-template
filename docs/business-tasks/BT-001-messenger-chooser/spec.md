## Что делает

Система связывания клиента в мессенджере (Telegram / MAX) с его контактом и сделкой в amoCRM. Три сценария входа:

1. **По contact_id** — менеджер отправляет клиенту ссылку с contact_id
2. **По телефону** — клиент переходит со страницы сайта/марквиза с номером в URL
3. **Мини-лендинг** — клиент с рекламы вводит телефон на лендинге и выбирает мессенджер

## Триггер

- **Веб-страницы:** GET `/messenger` и GET `/landing` в services/web
- **N8N:** amoCRM webhook на message events → entry-роутер → BT-001

## Логика

### Веб-страницы

| Роут | Параметры | Описание |
|------|-----------|----------|
| GET /m | cid | Страница выбора мессенджера, deeplink с contact_id |
| GET /m | p, us, um, uc, ut, uo | Страница выбора мессенджера, deeplink с телефоном + UTM |
| GET /l | us, um, uc, ut, uo (опц.) | Лендинг с полем ввода телефона + выбор мессенджера |

Deeplink форматы:
- TG: `https://t.me/vsc_strahovka_bot?start=PAYLOAD`
- MAX: `https://max.ru/id525708754339_bot?start=PAYLOAD`

Payload:
- По contact_id: `tgcid{ID}` / `maxcid{ID}`
- По телефону: `tel{PHONE}-utm_source-...-utm_content` (TG, разделитель `-`) / `tel{PHONE}_utm_source_..._utm_content` (MAX, разделитель `_`)

### N8N воркфлоу

| Условие | Действие |
|---------|----------|
| Deeplink содержит `cid` | Парс contact_id → найти последнюю сделку → привязать неразобранное → задача → чекбокс на контакте |
| Deeplink содержит `tel`, контакт найден, есть сделки | Привязать к последней сделке → задача → чекбокс на контакте |
| Deeplink содержит `tel`, контакт найден, нет сделок | Создать сделку → привязать → UTM → задача → чекбокс |
| Deeplink содержит `tel`, контакт не найден | Обновить телефон → UTM на сделке → задача → чекбокс |
